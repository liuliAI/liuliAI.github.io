Datawhale AI夏令营第四期：从零入门多模态大模型数据合成

天池Better Synth多模态大模型数据合成挑战赛 是 Datawhale 2024 年 AI 夏令营第四期的学习活动——

  • 适合想 入门并实践 多模态大模型、数据处理、模型训练 的学习者参与

温馨提示：本赛事需要耗费大量的算力，跑通此baseline至少需要120G内存以上的A10，整个流程可能需要5小时，大家请自备算力，预留好时间，下面我们会以阿里云为例。

我是在自己服务器上部署的，如果使用阿里云，创建镜像（非常简单，只需下载模型和数据集即可，环境已经配置好）参考以下教程即可https://linklearner.com/activity/14/13/27

以下讲解自行安装和部署遇到的报错和解决方法

Step1：下载赛事所需文件包！

包括：baseline、必要的开发者套件、基础模型 和相关数据集等，这是主办方提供统一的开发套件 dj_synth_challenge.tar.gz，参赛者下载解压后会得到如下文件目录： ![](/image/datawhile3.png) 复制如下命令，在终端中执行： `git clone https://www.modelscope.cn/datasets/Datawhale/better_synth_challenge_baseline.git`

Step2：安装训练和评测工具所需依赖！

推荐使用 conda 安装 
conda create -n dj python=3.10
conda activate dj

bash install.sh
这个时候，可能会有很多错误，我遇到以下错误： Failed to build installable wheels for some pyproject.toml based projects (pypi-kenlm) 括号中的可能不是`pypi-kenlm`，也可能是其他的，例如`simhash-pybind`，同类型错误都可以用以下方法解决，安装的包换个名字就行 解决方法①：下载“pyproject.toml-based，使用命令 `pip install pyproject` 解决方法②：安装gcc编译工具，请以root用户或具有sudo权限的用户运行以下apt命令： 
sudo apt update
sudo apt install build-essential
运行以下的gcc/g++命令打印GCC版本，验证GCC编译器是否已成功安装： `gcc --version` 或者`g++ --version` 解决方法③：更新工具包 `pip install --upgrade pip setuptools ` 最后在单独安装kenlm，使用命令 `pip install pypi-kenlm` 安装flash-attn失败 由于GPU型号、网络环境等原因，安装容易失败。 推荐源码安装： 1.看机器是否支持2.x版本 FlashAttention-2 currently supports: Ampere, Ada, or Hopper GPUs (e.g., A100, RTX 3090, RTX 4090, H100). Support for Turing GPUs (T4, RTX 2080) is coming soon, please use FlashAttention 1.x for Turing GPUs for now. Datatype fp16 and bf16 (bf16 requires Ampere, Ada, or Hopper GPUs). All head dimensions up to 256. Head dim > 192 backward requires A100/A800 or H100/H800. 机器支持2.x，下载main ,https://github.com/Dao-AILab/flash-attention/tree/main 机器不支持2.x，下载1.x ,https://github.com/Dao-AILab/flash-attention/tree/v1.0.9 下载完成后使用以下命令 `python setup.py install`自动安装 [ERROR] The global batch size of pretraining stage is not 256! Please check and retry. 看注释![](/image/datawhile4.png) 
apt update
apt install axel zip file
pip install modelscope

Step4：下载模型和相关数据集！

`bash download.sh`

Step5：下载BLIP图片描述模型！

新建一个python文件 ，如`a.py`，内容如下 
from modelscope import snapshot_download

model_dir = snapshot_download('goldsj/blip2-opt-2.7b', 
                              cache_dir='/mnt/workspace/better_synth_challenge_baseline/models', 
                              revision='master')
注意：mnt/workspace修改为自己的实际目录，例如我的是/data/ljq 命令行使用命令`python a.py`运行a.py下载BLIP图片描述模型

Step6：数据处理与合成！

在命令行执行dj-process --config solution/image_captioning.yaml
如果报错，说明上述环境没有安装好

Step7：执行模型训练 & 推理测评！

如果执行过 bash install.sh，则无需执行 git clone https://github.com/modelscope/data-juicer.git命令，否则下载
如果遇到连接超时报错，可以更换镜像源，https://github.com.cnpmjs.org/，这个网站将 Github 的资源进行了镜像，在国内访问速度很不错。

以下是每次执行git操作手动更换

git clone https://github.com/modelscope/data-juicer.git

修改克隆的项目地址，在 github.com 后添加 .cnpmjs.org

git clone https://github.com.cnpmjs.org/modelscope/data-juicer.git

也可以通过config进行全局替换

git config --global url."https://github.com.cnpmjs.org/".insteadOf https://github.com/

最后，可以训练和推理了，使用以下命令

cd better_synth_challenge_baseline/toolkit
bash train_mgm_2b_stage_1.sh

Step8：打包构建符合提交要求的zip文件！


cd submit
cp -r /mnt/workspace/better_synth_challenge_baseline/solution .
cp -r /mnt/workspace/better_synth_challenge_baseline/output/eval_results output/eval_results/
cp -r /mnt/workspace/better_synth_challenge_baseline/output/train.sh output/
cp /mnt/workspace/better_synth_challenge_baseline/output/training_dirs/MGM-2B-Finetune-default/finetuning.log output/training_dirs/MGM-2B-Finetune-default/
cp /mnt/workspace/better_synth_challenge_baseline/output/training_dirs/MGM-2B-Pretrain-default/pretrain.log output/training_dirs/MGM-2B-Pretrain-default/
zip -r submit.zip solution output

注意更换自己的路径

最后在官网提交结果(submit.zip文件)就行了
